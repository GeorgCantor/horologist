StreamlineSignInDefaultScreen

@Composable()
fun StreamlineSignInDefaultScreen(onSignedInConfirmationDialogDismissOrTimeout: (account: AccountUiModel) -> Unit, onNoAccountsAvailable: () -> Unit, columnState: ScalingLazyColumnState, viewModel: StreamlineSignInDefaultViewModel, modifier: Modifier = Modifier, content: @Composable() () -> Unit = { })
Content copied to clipboard

An opinionated implementation of StreamlineSignInScreen that:

  • displays the SignedInConfirmationDialog when there is a single account available;

  • displays the SelectAccountScreen when there are multiple accounts available, then displays the SignedInConfirmationDialog after the account is selected;

The content of this composable would be displayed when the screen is in "loading" state. This is an optional param, in case of no other layout is expected to be displayed while this screen is loading, e.g. in the scenario where the app is already displaying the splash screen.
